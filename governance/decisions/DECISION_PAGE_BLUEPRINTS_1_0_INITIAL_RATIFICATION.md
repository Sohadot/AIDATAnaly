## 1. Decision Summary

This decision ratifies `PAGE_BLUEPRINTS.md` as the governing page production standard for AIDAtanaly.

The Page Blueprints standard converts the ratified doctrine, thesis, ATI Standard, TFO Ontology, Route Map, Interface Governance, and Scanner Model into controlled public page templates.

This decision establishes how AIDAtanaly pages must be produced so the asset does not degrade into generic SEO content, thin reference pages, inconsistent layouts, broken link structures, or ungoverned marketing pages.

The governing thesis remains:

> Stages are states. Value lives in transitions.

Page Blueprints v1.0 is ratified as the production layer for building AIDAtanaly's public reference surface.

- category definition,

## 5. Minimum Viable Preview Clarification

This decision explicitly clarifies the meaning of the Minimum Viable Preview rule in `PAGE_BLUEPRINTS.md`.

A Minimum Viable Preview is not a substitute for the first public indexed reference release.

It may exist only as:

- private preview,
- internal review,
- staging environment,
- soft launch with `noindex`,
- or controlled non-indexed demonstration.

The first public indexed reference release remains governed by `ROUTE_MAP.md`.

Therefore, the first public indexed reference release must include the full 41-route Required Launch set.

The Minimum Viable Preview rule exists only to preserve coherence during internal or non-indexed review.

It does not authorize an incomplete indexed launch.

---

## 8. Measurement Gap Page Ratification

This decision ratifies the Measurement Gap page as a diagnostic constraint page, not a normal failure mode page.

The Measurement Gap page must state:

> Measurement Gap limits diagnostic confidence. It does not automatically prove that the transition is broken.

It must also state:

> If a vector is E0, Scanner v1 must not issue a numeric vector score, and if one or more vectors are E0, Scanner v1 must issue a Partial Profile rather than a clean composite ATI score.

This preserves consistency with `SCANNER_MODEL.md` and `DECISION_SCANNER_MODEL_1_0_INITIAL_RATIFICATION.md`.

---

## 9. Scanner Result Blueprint Ratification

This decision ratifies the scanner result view blueprint.

If all vectors are scorable, the result may display:

```
Composite ATI: {score} — {diagnostic_class}
Composite Evidence Confidence: {confidence_summary}
```

If one or more vectors are E0, the result must display:

```
ATI Profile: Partial
Scorable vectors: {list}
Unscorable vectors: {list} / E0 Absent Evidence
```

A clean composite ATI score is prohibited when any vector is E0.

## 11. Internal Linking Rule

This decision ratifies the universal internal linking model.

Every public page must link:

- upward to its governing parent,
- sideways to related concepts,
- and downward to governed child pages where applicable.

No page may function as an orphan.

No page may function as a dead end unless explicitly classified as a legal or terminal trust page.

Every scanner output must resolve into canonical reference architecture.

---

## 12. Interface Compliance

Every page produced under `PAGE_BLUEPRINTS.md` must comply with `INTERFACE_GOVERNANCE.md`.

Pages must preserve:

- Flow Made Visible,
- quiet states and weighted transitions,
- semantic motion only,
- score and Evidence Confidence separation,
- E0 as Unscorable,
- diagnostic class honesty,
- reduced-motion support,
- and readability without JavaScript.

A visually impressive page that weakens governed meaning is not compliant.
