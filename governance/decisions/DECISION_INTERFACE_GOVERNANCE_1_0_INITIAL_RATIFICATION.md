## 1. Decision Summary

This decision ratifies `INTERFACE_GOVERNANCE.md` as the governing interface standard for AIDAtanaly.

The interface standard establishes how AIDAtanaly must visually and interactively express its governing thesis:

> Stages are states. Value lives in transitions.

This decision confirms that the interface is not a decorative layer.

The interface is part of the asset's category logic.

AIDAtanaly's public surface must therefore embody movement, transition health, evidence confidence, diagnostic classification, and governed reference structure.

---

## 2. Ratified Interface Thesis

The ratified interface thesis is:

> Flow Made Visible

This means the interface must make transition movement legible.

The visual system must prioritize:

- transitions over isolated stages,
- movement over static volume,
- diagnostic clarity over decoration,
- evidence separation over false certainty,
- governed components over arbitrary layouts,
- and accessibility, performance, and SEO safety over visual spectacle.

The interface must not merely make AIDAtanaly look modern.

It must make the AIDAtanaly thesis understandable.

---

## 3. Governed Visual Principle

The ratified visual principle is:

> States are quiet. Transitions carry the weight.

AIDA stages may appear as stable positions, nodes, or states.

The transitions between them must carry the primary visual emphasis.

Therefore, the canonical interface motif is the transition axis:

```
Attention → Interest → Desire → Action → Loyalty
```

The axis must be used as a governing visual structure across:

- homepage hero,
- vector pages,
- scanner experience,
- scanner outputs,
- diagnostic summaries,
- and selected category pages.

Any interface pattern that visually overemphasizes isolated stages while weakening transition movement violates this standard.

## 5. Evidence Confidence Visual Separation

This decision ratifies the visual separation of ATI score and Evidence Confidence.

The ATI score measures transition health.

Evidence Confidence measures reliability of the score.

The interface must never visually merge the two.

Required rule:

> Score visuals and Evidence Confidence badges must remain distinct.

Evidence Confidence badges must use neutral visual treatment.

They must not use the diagnostic color scale.

They must not make a weak-evidence score appear stronger or weaker than the actual score.

E0 must be displayed as:

```
Unscorable / E0 Absent Evidence
```

E0 must not display a numeric score.

This ratifies the interface implementation of ATI Standard governance rules 7 and 8.

---

## 6. Diagnostic Color Governance

The interface may use a governed diagnostic scale for ATI classes.

The diagnostic visual scale must map exactly to the ATI diagnostic classes:

| Diagnostic Class | Score Range |
|---|---|
| Transition Strong | 85–100 |
| Transition Functional | 70–84 |
| Transition Unstable | 55–69 |
| Transition Constrained | 40–54 |
| Transition Critical | 0–39 |

A score must never be visually exaggerated.

A score of 48 must not appear visually equivalent to a score of 85.

A constrained transition must not be styled as healthy.

A critical transition must not be softened in a way that hides diagnostic severity.

The interface must preserve measurement honesty.

---

## 7. Motion Governance

Motion is permitted only when it explains meaning.

Permitted motion includes:

- transition flow,
- friction slowdown,
- drop-off interruption,
- signal decay,
- diagnostic reveal,
- scanner progression,
- and controlled state-to-state movement.

Prohibited motion includes:

- decorative particles,
- random parallax,
- scroll-jacking,
- carousel motion used for spectacle,
- gaming-style animation,
- excessive hover effects,
- and motion that distracts from diagnosis.

The interface must respect `prefers-reduced-motion`.

Reduced-motion users must receive a static equivalent that preserves the same meaning.

Motion may enhance the thesis.

Motion must not carry the thesis alone.

---

## 8. WebGL and 3D Rule

WebGL, 3D, or spatial interface elements are prohibited by default.

They may only be introduced by a future governance decision if they pass all of the following tests:

1. The element explains a governed concept.
2. The page remains understandable without it.
3. It does not weaken SEO.
4. It does not break mobile performance.
5. It does not harm accessibility.
6. It does not make the asset look like a game.
7. It improves trust rather than spectacle.
8. It supports the thesis of transition movement.

No WebGL or 3D element may be added as decoration.

## 11. Accessibility and Performance Commitment

The interface must be accessible, performant, and SEO-safe.

Ratified requirements include:

- semantic HTML,
- readable typography,
- visible focus states,
- keyboard navigability,
- sufficient contrast,
- reduced-motion support,
- no content locked behind JavaScript,
- reference pages readable without JavaScript,
- controlled asset weight,
- stable layout,
- and no unnecessary visual dependencies.

Performance targets:

| Metric | Target |
|---|---|
| LCP | < 2.5s |
| CLS | < 0.1 |
| INP | < 200ms |

The interface must support WCAG 2.2 AA as the target accessibility posture.

## 12. SEO and Agent-Readable Interface Rules

The interface must not hide governed meaning inside decorative layers.

All primary definitions, diagnostic labels, stable IDs, route relationships, and scanner explanations must be available in semantic HTML.

Future `/data/*.json` files must match public interface language.

The interface must remain legible to:

- humans,
- search engines,
- AI agents,
- assistive technologies,
- and strategic reviewers.

A visually impressive interface that obscures meaning is not compliant.
