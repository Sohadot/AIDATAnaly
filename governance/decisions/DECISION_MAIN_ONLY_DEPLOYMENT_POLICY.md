## 1. Decision Summary

AIDAtanaly adopts a **main-only source** deployment model.

`main` is the sole development, governance, and release source branch. Public website exposure is produced from a governed `dist/` package built on `main` and published through **GitHub Actions** to GitHub Pages.

The `gh-pages` branch is **retired** as a deployment mechanism.

---

## 2. Ratified Rules

### 2.1 Source branch

- **`main` is the only branch** for implementation, governance, data, scripts, and release posture.
- No permanent development branches.
- No permanent deployment branches.

### 2.2 Deployment package

- **`dist/`** remains the governed public website package.
- `dist/` is generated by `scripts/build-dist.ps1` from `main`.
- `dist/` stays in `.gitignore` and is **not committed** to the repository.
- Repository root must **never** serve as the public website root.

### 2.3 GitHub Pages source

- GitHub Pages **Build and deployment → Source** must be **GitHub Actions**, not **Deploy from a branch**.
- The workflow `.github/workflows/pages.yml` on `main` is the sole authorized public deployment path.

### 2.4 Workflow requirements

The Pages workflow must, on each authorized run:

1. Check out `main`.
2. Run `scripts/quality-gate.ps1 -IndexedRelease`.
3. Upload the **`dist/`** directory as the Pages artifact (after the gate builds and validates it).
4. Deploy the artifact with `actions/deploy-pages`.

No workflow may publish repository root, Markdown governance files, `governance/decisions/`, `scripts/`, or `preview/`.

### 2.5 Local operator script

- `scripts/deploy-dist.ps1` is a **local preflight** helper only (build + validate + policy check).
- It must **not** push to `gh-pages` or any other deployment branch.

### 2.6 gh-pages retirement

- After GitHub Actions deployment is verified live, the operator may delete the legacy `gh-pages` branch.
- Deletion must occur **only after** successful Actions-based Pages deployment is confirmed.

## 3. Relationship to PUB-REL-001

PUB-REL-001 remains ratified for:

- 41-route release requirement,
- `dist/` as deployment package,
- repository root prohibition,
- indexation activation procedure,
- exposure boundary (dist artifacts only),
- claim and messaging boundaries.

PUB-REL-002 **amends** only the deployment transport:

| PUB-REL-001 (prior) | PUB-REL-002 (now) |
|---------------------|-------------------|
| `gh-pages` from `dist/` allowed | `gh-pages` **retired** |
| Branch-based Pages publish | **GitHub Actions** artifact publish |
| `deploy-dist.ps1` pushes `gh-pages` | `deploy-dist.ps1` local preflight only |

Operator decisions **A**, **C**, **D**, and **E** from PUB-REL-001 §21 remain in force.

Operator decision **B** is superseded by this decision.

---

## 4. Operator Actions Required

1. GitHub → **Settings → Pages → Build and deployment → Source: GitHub Actions**.
2. Merge workflow to `main` and confirm a successful **Deploy GitHub Pages** workflow run.
3. Verify `https://aidatanaly.com/` (or the repository Pages URL) serves the `dist/` artifact.
4. After verification, optionally run: `git push origin --delete gh-pages`.

---

## 5. Non-Goals

This decision does not:

- change ATI, TFO, scanner scoring, or route map,
- revert indexation posture,
- commit `dist/` to git,
- expose governance decision logs as website routes.
