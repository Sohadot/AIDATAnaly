Governance

AIDAtanaly is a governed reference system. This page explains how standards, routes, claims, and interfaces stay disciplined before and after public release.

Why Governance Exists

Diagnostic products fail trust when scores, failure names, and recommendations drift without control. Governance keeps ATI, TFO, the scanner, and public language aligned to ratified documents so users can trace every output to a governed definition.

Governed Documents

Core ratified documents include ATI Standard, TFO Ontology, Scanner Model, Route Map, Page Blueprints, Interface Governance, and Implementation Plan. Machine-readable registries under /data/ mirror these documents for validation and scanner behavior.

  • ATI Standard - scoring and Evidence Confidence separation
  • TFO Ontology - failure mode taxonomy
  • Scanner Model - rules-governed diagnostic engine (v1)

Versioning Principle

AIDAtanaly uses sequential document versioning. Change severity is recorded in decision logs and is not encoded directly into the version number.

Public pages reference ratified versions (for example ATI v1.1, TFO v1.0, Scanner v1.0). Material changes require governed review before the site presents them as current.

Decision Logs

Significant ratification and separation decisions are recorded in internal decision logs. Severity and rationale are preserved for auditability. Public pages summarize outcomes; detailed decision artifacts remain internal unless a future release plan explicitly publishes them.

Claim Restraint

AIDAtanaly must not present diagnostic output as revenue proof, industry adoption, or guaranteed improvement. Prohibited patterns include fear-based copy, fake urgency, and language implying the system knows exact commercial causality. See Sources for adoption posture.

Evidence Confidence

Evidence Confidence is never a score component. E0 vectors are Unscorable. Partial Profile rules apply when any vector lacks evidence. This separation is non-negotiable in standards, data, interface, and scanner output.

Route Governance

Every public page maps to a route in the governed Route Map. Required Launch routes must exist before sitemap inclusion. Deferred routes may be linked as planned references but must not be presented as live product surfaces until implemented.

Interface Governance

Visual components separate diagnostic scores from Evidence Confidence badges, represent E0 as Unscorable (never numeric), avoid gamified score effects, and support reduced motion. No external JS, analytics, GPU/3-D rendering, or decorative animation libraries on reference surfaces.

Scanner Governance

Scanner v1 reads rules from JSON registries only. Failure mode IDs, intervention layers, and canonical routes must match TFO and Route Map. Results stay on-page; dynamic result URLs must not become indexable routes without separate governance.

Public Limits

  • Diagnostic references, not commercial guarantees.
  • Class-level intervention recommendations unless evidence supports more specificity.
  • No paid ranking of tools or generic affiliate monetization on reference pages.
  • Reports and briefs may be offered without hiding basic definitions behind payment.
